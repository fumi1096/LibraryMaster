# RAG 查询 API 接口文档

## 概述

基于 FastAPI 构建的 RAG（Retrieval-Augmented Generation）查询接口，使用 LanceDB 作为向量数据库，提供图书信息的向量搜索功能。

## 服务信息

- **服务地址**: http://localhost:9014
- **数据库路径**: /data/lancedb
- **数据表**: books

## 接口详情

### 1. 健康检查接口

**GET** `/`

**响应示例**:
```json
{
  "message": "RAG Query API is running. Use /query endpoint for vector search."
}
```

### 2. 列出数据表接口

**GET** `/tables`

**响应示例**:
```json
{
  "tables": {
    "tables": ["books"],
    "page_token": null
  }
}
```

### 4. 向量查询接口

**POST** `/query`

**请求参数**:
```json
{
  "vector": [1024个浮点数],
  "count": 5
}
```

**字段说明**:
- `vector`: 查询向量，必须是 1024 维的浮点数数组
- `count`: 返回结果数量，默认为 5，必须大于 0

**响应示例**:
```json
[
  {
    "书名": "甲古文化与中医学",
    "作者": "李良松，刘学春著",
    "出版社": "北京中国中医药出版社",
    "关键词": "string",
    "摘要": "string",
    "中国图书分类号": "string",
    "出版年月": "string",
    "similarity": 1023.7447509765625
  }
]
```

**响应字段说明**:
- `书名`: 图书名称
- `作者`: 作者信息
- `出版社`: 出版社信息
- `关键词`: 图书关键词
- `摘要`: 图书摘要
- `中国图书分类号`: 图书分类号
- `出版年月`: 出版年月
- `similarity`: 相似度分数（实际为距离，值越小越相似）

## 错误响应

所有接口在出错时都会返回相应的 HTTP 状态码和错误信息：

- **400**: 请求参数错误
- **500**: 服务器内部错误

## 使用示例

### Python 请求示例

```python
import requests
import numpy as np

# 基础URL
API_BASE_URL = "http://localhost:9014/rag"

# 向量查询示例
query_vector = np.random.randn(1024).tolist()
query_data = {
    "vector": query_vector,
    "count": 3
}

response = requests.post(
    f"{API_BASE_URL}/query",
    json=query_data,
    headers={"Content-Type": "application/json"}
)

if response.status_code == 200:
    results = response.json()
    for result in results:
        print(f"书名: {result['书名']}")
        print(f"作者: {result['作者']}")
        print(f"出版社: {result['出版社']}")
        print(f"相似度: {result['similarity']}")
        print("-" * 50)
else:
    print(f"请求失败: {response.status_code} - {response.text}")
```
